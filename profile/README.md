# iHexHubs

**Research and technical work on independently verifiable evidence for high-stakes software systems.**

iHexHubs explores how organizations can produce portable, independently verifiable evidence for financial and administrative decisions.

The technical approach combines:

> Canonical records → signed receipts → versioned rules → recomputable witnesses → independent verification

## What it explores

- Deterministic records and stable hashing for auditable software events.
- Signed receipts that can be verified outside the original system.
- Versioned policy bundles for reproducible decisions.
- Recomputation of decision traces through an independent verifier.
- Evidence designed to support audit, compliance, dispute review, and traceability.

## Relationship to SixDriven

iHexHubs was the technical work that led to SixDriven.

While designing systems that require strong evidence, traceability, and reproducible decisions, I identified a broader software-delivery problem: AI-assisted development needs reliable connections between intent, rules, implementation, review, and integration.

That insight became SixDriven, the current primary project.

## Current status

iHexHubs is under active research and technical development.

It does not claim production deployment, external adoption, or validated commercial outcomes at this stage. Future work includes controlled evaluation, reproducible testing, and pilots with appropriately protected data.
